欢迎使用Terminal IDE。

前言

Terminal IDE是一个基于命令行拥有许多应用程序的可扩展系统，这些程序完全运行在Terminal IDE主程序的空间之中内。
Terminal IDE工作在Android系统的正常权限下，因此您不需要为其获取Root权限或者其他任何特殊的权限。

连接

通过登录telnet或者ssh来使用您的大尺寸外接键盘配合外接显示器，能够最大化的发挥Terminal IDE的功能!当然了，utelnetd和sshd也是可用的，它们是使用本应用的首选方式。

JAVA

Terminal IDE还拥有一个完整的JAVA的开发环境，您可以用它来构建Java命令行程序或者完整的Android应用程序。在~/system/src/目录下有一组示例程序展示了如何构建和运行它们。

多亏了BusyBox使得:bash, vim, javac, java, dx, aapt, apkbuilder, signer等等都可以在命令行下面正常运行。

键盘

因为Android系统原本的设计使用的键盘没有Ctrl,Esc,Alt等键码，鉴于此，Terminal IDE提供了一个完整的ANSI的键盘来胜任这个工作。

您必须在Android系统中的设置—->语言和时间设置中勾选Terminal IDE键盘，您才能在Terminal IDE中使它。
通过它，您会得到一个完整的带有Tab,Ctrl,Alt,Del等功能键的ANSI键盘。

兼容性

一个已知比较重要的问题是主机名解析。由于静态链接的C语言二进制库，在没有Root的手机上将没有/etc文件夹。在Terminal IDE中，除了C程序，主机名解析一般不工作。使用jping获取ip地址和主机名的时候，一些主机名您需要使用“数字”的ip地址而不是“字符串”的主机名。C语言程序将会正常工作而不受这个限制。(例如:weget,git,mc等)

# jping www.google.com

执行以上命令，您将会获得Google的IP地址。

其它

由于SSL协议没有被内置在大多数APOS标准的Android系统中，因此您可以使用SSH协议，并且通过SSH来创建代理管道。
