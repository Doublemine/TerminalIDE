键盘

键盘对于Terminal IDE非常重要！您需要通过键盘上的这些控制功能键来确保命令行程序正确的运行。
Terminal IDE附带了一个完整的128组键码的键盘。

注意：

  • Terminal键盘仅仅只有在Terminal IDE或者其他终端中才能真正的完美工作。

这就是为什么你们当中有一部分人抱怨回车键不工作，它仅仅正常工作在Terminal IDE中而不是Android系统所有应用中。

你如果能在键盘上发现发送(send)键,下一个(next)键或者确认(go)键的图标，那么它们将能够在Android系统的其他应用中正确的充当回车键。

当你在锁屏状态下需要输入PIN码然后输入回车键来解锁的时候，你可以使用Ctrl+Enter键的组合来完成这个操作，这时它能够正常的充当回车键。

你需要在你的设备的系统设置>语言与时间设置选项中勾选Terminal IDE键盘才能启用它。

你可以在程序的主界面中的键盘选项中切换到Terminal IDE键盘，通过Terminal IDE键盘右下角的按钮，你也可以轻松的切换到其他键盘。

在Terminal IDE右下角的键盘设置中，你可以根据你的屏幕大小设置适合你的键盘大小。

Terminal IDE还附带了一个强大的硬件键盘映射，因此绝大多数外接键盘都应该可以正常工作。
你还可以在键盘设置选项中设置设备的物理按键来充当Ctrl键和Esc键。

Ctrl键是至关重要的一个键:

ESC = CTRL+[
    [ALT+X] = [先按下ESC 然后再按 X]

为了查看键盘的组合键，在Bash中，按下Ctrl+V键之后，然后按下任意键，将会在屏幕上显示它接受到了什么键码。

为了正确设置诸如蓝牙键盘等外接的硬件键盘，你需要使用硬件键码映射。该功能在软件住界面中的设置>外接键盘映射选项中。

在设置之前，你应该检测有哪些键码正常工作，哪些工作不正常。绝大多数情况下，这些设置都会自动完成。

功能键是一个常用的键，因此，你必须在你的键盘上选择一个键来充当功能键Fn。

首先，在终端界面中，通过菜单选项打开选项开关>键值记录器，它默认是关闭的。这将会输出你按下的键码的键值，具体信息保存在~/.keylog文件中。

然后你可以在终端中输入以下命令来查看:

# tail -f ~/.keylog

按下Ctrl+C退出查看。

这个操作将会poll到tail命令的标准输出，每0.5秒钟刷新一次，所以有一点轻微的延迟。现在，按下键盘上的一些键码，得到的键值将可用于外接键盘映射部分的设置。

键值记录器的输出也会发送给LogCat，你也可以通过查看LogCat来获取键值。

如果你按下一个键却没有接收到响应的键值，比如Fn键。那是因为这种类型的按键本身没有任何有效的键码，他们必须配合其他有效的键码来组合完成特定的功能，才能被键值记录器捕获到：

  • 只要它能够产生键值，你可以任意映射你键盘上的键码。
  • 不要忘记关闭键值记录器当你使用完毕的时候。

在外接键盘的映射的顶部，你可以设置启用或者禁用键盘映射。如果你覆盖了一些系统按键，然后你打算重置这些键码，只需要将这些键值设置为-1即可。只允许整形数值。

我使用的索尼大法的蓝牙键盘通过这个方法完美运行~码字就像吃了炫脉，根本停不下来。。
